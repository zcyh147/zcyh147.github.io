人人都能用 WAAPI（四）“元工具”：WAAPI Code Generator
2021-01-15
字数总计: 2.2k|阅读时长: 7 分钟

大家好，我是溪夜。
在《人人都能用 WAAPI》第四篇中会分享一个用于提升 WAAPI 开发效率的程序，作为一名从小就接触了编程的人，我始终认为使用代码优化工作中的任何重复都是非常必要的，这也是 WAAPI Code Generator 开发的最初愿景。
这个程序的设计理念与本系列文章的主题《人人都能用 WAAPI》颇为接近，是一个编写 WAAPI 代码的“元工具”。我设计了一种优雅且一站式的编码方案，以此提高 WAAPI 使用时的效率。
除了对程序的使用方法进行介绍外，本文还会对程序内涉及 WAAPI 的代码进行简介，以方便大家参考在生产环境中的代码用法。

对现有 WAAPI 编码流程的思考

在使用 WAAPI 时很容易遇到这样的痛点：

  1. 面对 ak.wwise.core.object.get 或 ak.wwise.core.object.create 这种选项繁杂的 API，手写 JSON 格式的配置代码块时体验不是很好，一不小心写错了符号或缩进很容易程序就无法正确执行了。
  2. 无论是否参考第一篇中曾提到的分类逻辑，在文档中搜寻想要的 API 时都要翻阅一番。
  3. 获取 GUID、Path 等参数时，首先需要切回 Wwise 中，按住 Shift 再右键点击对象后，在弹出的菜单中才能找到拷贝选项。

在“能偷懒则偷懒”这条核心逻辑的驱使下，我在思考能否进行一类元工具的开发。就像重复代码块写作时会用到 Code Snippets 一样，尽量减少重复工作带来的时间浪费。
这也是 WAAPI Code Generator 诞生的动机，我的初衷就是希望把 WAAPI 的使用变的更加简单且工具化。

设计过程

2.1. 起因：如何进一步偷懒？

以 ak.wwise.core.object.get 为例，在任何编程语言中调用这个 API 需要以下步骤：

  1. 根据文档选定需要使用的参数，例如 form 决定从何处获取数据，transform 决定获取数据的变换条件，此外还有可选参数 Options 决定返回值及平台等。
  2. 对于form 来说，需要手动获取 GUID 或 Wwise 工程中的路径参数，而其他可能涉及的参数也得哦按需从文档中获取。
  3. 书写 JSON 语法的配置信息，把对应的参数放到 JSON 那层叠括号中的正确位置里。
  4. 测试代码。（如果 JSON 格式的配置一不小心写错了点什么，恭喜你，愉快的 Debug 开始了！）

而在 WAAPI Code Generator 中你只需要：

  1. 确保已连接到 WAMP 服务器，在左边第一栏中间选择需要的 WAAPI 分类。例如在本例中需点选 (Functions) Object Operation 类，之后即可轻松在下面的 API 列表中找到 ak.wwise.core.object.get。
  2. 在程序中间第二栏的下拉列表里选择所需的参数。此时右边会有官方文档及我所书写的文档注释。如果需要使用 GUID 或工程路径等信息作为参数，可通过程序中复刻的 Wwise Project Explorer 查看当前工程的完整架构，点击 Add 即可把所需参数添加到代码中。
  3. 选中所需的编程语言，点击 Generate，即可自动生成当前语言对应的代码。默认情况下生成的所有代码都会自动复制到剪贴板，如果需要代码中一部分，也可点击对应的按钮选择所需的部分。

容易看出，这个程序中优化掉了 WAAPI 使用中繁琐的步骤，例如需反复的翻阅文档、费心书写 JSON 格式参数、去 Wwise 内拷贝所需数据等操作。

2.2. 目的：解放记忆力并实现快速开发

有一句流行语“懒惰是第一生产力”，说的正是通过技术的进步，降低了用户获得最终结果的成本。
这也是我开发 WAAPI Code Generator 的首要动机，通过解放记忆力把 WAAPI 工具化，将更多精力投入到工具的逻辑架构上。
但只有脚本或后端代码是远远不够的，不仅其他非程序出身成员的使用体验非常糟糕，我自己也无法忍受没有图形界面带来的缺失感。GUI 作为现代计算机与人类之间沟通的桥梁，交互体验无疑是特别重要的一部分。

连接 “GUI Gap”（图片来源于 PySimpleGUI 介绍文档）
如何解决前端问题呢？
工业级软件生产中常见的桌面平台开发框架有使用 C# 的 WPF 或 UWP，使用 C++ 的 Qt（Python 也可以使用 PyQt），基于 Web 为代表的 Electron 等等。关于各种框架的性能、打包体积、跨平台等优缺点在这里就不展开讨论了，但无疑都存在一个问题，上手相对较慢，开发流程不够简单直接。

WAAPI Tool 量产架构图
WAAPI Code Generator 是使用 PySimpleGUI 开发的，也是我个人对此框架开发潜力的探索尝试之一。它的优点是简单易用、开发效率快、跨平台、UI 设计语言审美还不错。当然缺点也有很多，例如大型生产环境中没见到任何人使用、macOS 下打包麻烦、无法实现 MVC、MVVM 等复杂架构等等。
但这并不妨碍它作为一个低开发成本 GUI 框架的存在，尤其在快速为 Python 脚本创建一个原型级的 GUI 时，别有一番优势。对简单逻辑的小程序而言，用熟了 PySimpleGUI 后在半小时内即可为你的脚本设计一套说得过去的交互及 GUI。
作为轻生产需求或 Demo 工具开发而言，这无疑是 Python 用户们最适用的一个入门 GUI 框架，在下一节中我会演示一下如何为 WAAPI Code Generator 生成的代码快速加上基于 PySimpleGUI 的 GUI 代码。

3.1. 下载与运行

GitHub 地址：https://github.com/zcyh147/WAAPI-Code-Generator

  1. 打开 GitHub 仓库，在 releases 中下载所需版本的压缩包，解压到你喜欢的目录中。仓库中根据系统与帮助文档的不同，总共分为四个版本，请注意压缩包的后缀。
  2. 根据 Wwise 内的 WAAPI 设置，填入程序中所设置的服务器和端口号来连接 WAAPI 服务器，没修改过设置的话直接点击 Connect 按钮即可。
  3. 本程序遵循 Don’t Make Me Think 原则，采用“左中右”三分栏的设计，用户从左到右进行顺序操作即可，非常简单。
