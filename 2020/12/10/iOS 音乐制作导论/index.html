iOS 音乐制作导论
2020-12-10
字数总计: 4.7k|阅读时长: 15 分钟

大家好，我是溪夜。
2011年我见到了 iRig（IK Multimedia 生产的第一代 iOS 端吉他音频接口），它直接颠覆了我对电脑音乐制作的认知。
对于一个当年的软件效果器用户来说，使用电脑版 Guitar Rig（IK Multimedia 出品的电脑版的软件吉他效果器）意味着需要笨重的笔记本电脑+声卡，但 iRig 竟然只需要一台 iPhone 甚至廉价的 iPod Touch 即可代替这些，结合 iOS 版 Guitar Rig 竟然还能提供说得过去的吉他音色。
自此，iOS 移动音乐制作领域的潜力就在我心中扎下了根。
在去年北京的模块合成器交流方式上，我结识了 Dancing K 老师（以下简称为 DK 老师）。作为前 Novation 大中华区代言人，他在 iOS 音乐制作领域深耕多年。聊了短短几分钟，就把我从只了解 GarageBand（库乐队）的水平，引导到了现代 iOS 音乐制作的技术领域前沿。原来这么多年来 iOS 平台上的音乐制作类软件茁壮成长，早已成长为可全功能对标桌面音乐制作平台的环境。
随着去年 iPad OS 与今年 Apple M1 芯片的发布，苹果生态系统下的移动端与桌面概念一再模糊，这也使 iOS 平台的潜力进一步提升。
基于在 iOS 移动音乐制作上的兴趣，我决定利用业余时间与 Dancing K 老师合作，共同推广移动音乐制作技术。
作为今年整理的 iOS 音乐制作相关知识的总结，我写一篇导论类文章与大家分享，希望能为移动音乐制作领域添砖加瓦。

类比理解 iOS 与桌面端音乐制作的区别

对于初次接触 iOS 音乐制作的朋友，我更推荐大家以类比的方法来理解移动端音乐制作。
桌面端音乐制作的核心抽离出来只有两点，即录音与后期处理。
不管是纯模拟，还是现今流行的纯数字或 Hybrid Mixing（数模混用混音），这两者都是必须要完成的核心步骤。
大家都对电脑音乐制作一般有着比较深的认知。通过相同概念的类比，可以更快的理清 iOS 平台的解决方案与电脑平台的区别。

2.1. 录音

比较简单的录音设备套装，需要话筒（录制声音）、音频接口（ADDA、话放）、监听设备（耳机与音响）三者，或者通过一个集成音频接口的话筒可代替前两者，再配一个监听设备即可。
如果选择第一种解决方案（话筒+音频接口），市面上现在几乎所有 USB 音频接口均可与 iOS 设备连接，如果碰巧音频接口和 iPad 都是 USB Type-C 接口的，两者可以不需要转接头实现直连。在声卡自带或第三方话放满足的情况下，理论上你能用任何想用的话筒进行录音。
如果选择第二种解决方案，恭喜你，现在市面上 iPad/iPhone 话筒有着大量优秀的解决方案，Apogee、Blue、Rode、Shure 等厂商都有着很棒的产品。
【配图】
Lighting 接口的直插话筒，适用于把 iOS 设备快速变成录音笔的使用场景
【配图】
专业的 USB 大振膜电容话筒，只需要连接到 iOS 上就能获得极佳的录音体验

2.2. 后期处理

对于混音来说，iOS 平台上你既可以用到与桌面平台体验完全一致的 Waves、FabFilter、Eventide 等大牌效果器，也可在其他丰富的效果器品牌（如 Nembrini Audio，ToneBoosters）中找到你想要的那一款。
【配图】
如果想要进行母带处理，也可以看到如 Positive Grid 出品的 Final Touch，IK Multimedia 出品的 Lurssen Mastering Console 等专业级母带工具。

硬件连接

无论是什么类型的音频设备系统，找准接口都是最重要的，在这部分我们一起看一下现在 iOS 设备的外设连接情况。
【配图】
需要使用 30-pin 接口（如 iPad 落地式效果器 DigiTech iPB-10）和使用 3.5 mm 耳机接口（如文初提到的 iRig）连接的 iOS 外设早已退出历史舞台，这类 iOS 设备本身也不在销售了，故这部分陈旧接口的设备在此不再提及。
在此向陪伴我12年的 iPod Classic 致敬，在更换了电池和 SSD 后它依然能够稳定的工作，持续的为我播放美妙的音乐。

3.1. Lighting 接口的 iOS 设备

对于采用传统的 Lighting（闪电）接口的设备，有一个配件必须要购买，即 Lightning to USB 3 Camera Adapter（闪电转 USB 3 相机转换器）。
【配图】
这个转接器能够为 iOS 设备拓展出一个重要的 USB 3 接口，并且保留一个 Lighting 接口供充电使用。
通过拓展出的 USB 3，我们就可以连接各种设备了。比如直接连接声卡，或者先连接 USB 拓展坞再通过拓展坞上的 USB 接口连接各种设备。

3.2. USB Type-C 接口的 iOS 设备

在比较新的 iPad 设备上，如 11 英寸 iPad Pro（第 1 代和第 2 代）和 12.9 英寸 iPad Pro（第 3 代和第 4 代）或 iPad Air 4，它们都采用了 USB Type-C 以提供更强大的兼容性。
对于这些 iPad，可以直接把支持 USB Type-C 的音频接口接到 iPad 上即可正常使用。
如果你有拓展坞，也可以尝试接到拓展坞上。不过我建议大家选择非直插式拓展坞，否则容易损害你的 USB 接口。此外建议事先对拓展坞的兼容性做好心理准备，如不兼容及时进行退货。

iOS 软件间的数据交换协议

在桌面平台中有着多样的音频插件协议，常用的为 VST、Audio Units、AAX，此外还有 REAPER 中的 JS，Pro Tools 陈旧的 RTAS、TDM 等。
随着 iOS 平台的发展，现在移动端也有了多种传输协议。

4.1. 插件协议

4.1.1. Inter-App Audio (IAA)

IAA 发布于 2013 年的 iOS 7 中，与前者 Audiobus 操作相同，主要的实现方法是应用之间的音频串流。
IAA 作为比较原始的音频串流协议，必须保证全屏运行才能进行正常工作，这也让其成为了一种比较原始正在被淘汰的技术。

4.1.2. Audio Unit v3（AUv3）

AUv3 发布于 2015 年，与桌面平台的 AU 协议相同，两者可以共用以节省双平台的开发时间。
作为真正的系统级插件协议，可以实现与桌面端相同的使用场景，即在应用中调用 AUv3 插件时可实现 Host 内运行，而无需打开新的窗口。
在进行 iOS 音乐制作时，AUv3 类插件是首选。
关于 IAA 与 AUv3 的区别，汤总有一篇文章讲解的很好：https://www.midifan.com/modulenews-detailview-33640.htm

4.2. 同步协议

4.2.1. Ableton Link

在多个乐器协作时候，MIDI CLock 的同步无疑是重要的需求。而 Ableton Link 的主要功能就是解决 MIDI CLock 不同步的问题。经过多年发展，目前除了有大量的软件提供支持，一些硬件也支持了 Ableton Link。
它既可通过有线网络进行同步，也可通过 WiFi 进行同步，使用场景很广泛。

iOS 如何与电脑端进行交换？

工程交换能力是我最看重的功能。
除了非常 Geek 的 Live Set 玩法及满足于 iOS 上的后期处理效果的普通用户外，还是有一部分人会思考能否对 iOS 上的工程后在桌面平台上进行完善。
答案是完全可以的。
我们都知道，iOS 平台纵使效果器与合成器算法再好，它也有天生的缺点：

  1. Host（DAW）类 App 大多无法做桌面级信号 Routing，这也让模拟硬件的加入相对比较困难。当然，把硬件效果器直接接到支持 Line in 移动音频接口上做一次性渲染也不是不行，但相信除了极少数 Geek 没人会这么做。
  2. iOS 平台天生的空间不足，让高质量（也需很大存储空间）的音源类 App 几乎难以存在。这也让移动端编曲时想得到高品质的乐器音源相对桌面平台来说难了很多，采样乐器靠空间带来的音质差距是很难弥补的。
  3. iOS 平台的处理能力终究有限，一般来说新款 iPad 的性能足够强大到让大家折腾，但万一 CPU 消耗大的 App 开启过多，性能还是会吃紧。当然，入门级用户不用考虑这个问题。

6.1. 传输工程文件

6.1.1. Garageband（库乐队）

因为 Garageband 同时具有 iOS 和 macOS 版，所以将 iOS 版的工程在 Mac 中打开是十分容易的。请注意，这两者只支持单向流动，因为 macOS 版具有一些移动端不具有的特性，所以 macOS 版的项目文件无法在 iOS 端打开。
在 Garageband 的“我的乐曲”浏览器中，选取需要传输的工程，分享为“项目”。
随后可以通过常见方式把文件传输到 Mac：

  • 连接 Mac，通过 iTunes 传输浏览到刚导出的 .band 格式的项目文件，并用 macOS 版 Garageband 打开编辑。
  • 通过 AirDrop 直接隔空投送刚产生的项目文件，或通过 iCLoud / Dropbox 等进行文件中转。

在 Mac 上打开 Garageband 后，因为兼容性的问题，iOS 端添加的所有效果器插件会消失。如果对特定的插件效果有需求，可以提前渲染轨道。
相关的信息可以参考苹果官方文档：https://support.apple.com/zh-cn/guide/garageband-ipad/chs39284d66/ipados

6.1.2. Korg Gadget 2

Korg Gadget 2 也同时具有 iOS 和 macOS 版本，因此我们也可以把 Gadget 2 的工程传输到 Mac 上进行编辑。对于 Gadget 2 来说，macOS 与 iOS 版本是互通的，因此只需要将文件存到 iCloud 上即可实现两者之间的无缝访问。
相关的信息可以参考 Korg 官方文档：https://support.korguser.net/hc/en-us/articles/115001317186-Syncing-with-Gadget-for-iOS
Gadget 也可以通过导出 .als 文件从而在 Ableton Live 中进行编辑。
除此之外，因为 Gadget 还有 Nintendo Switch 版，iOS 设备还可通过扫描二维码来获取Switch 上的工程。

6.1.3. Cubasis

对于很多拥有 iPad 但没有 Mac 的用户来说 Cubasis 才是最佳之选。一方面因为其拥有良好的原生 Waves 插件支持，另一方面也是因为 Cubase 支持 Windows 平台，不像上面两者为 Mac 独占。
对于 Cubasis，导出工程到电脑很简单：

  1. 选用 AirDrop 或任何你喜欢的文件分享手段，把工程从 iOS 设备中分享到电脑后会是 .zip 格式的压缩包，将其解压之后即可获得 .cpb 格式的 Cubase 项目。
  2. 在 Cubase 中点选 File -> Import -> Cubasis，浏览到之前的 .cpb 文件即可。

相关的信息可以参考 Cubasis 使用手册中的 FAQ 部分：https://www.steinberg.net/fileadmin/files/PRODUCTS/Apps/Cubasis/Cubasis_Help/FAQ.html

6.1.4. FL Studio Mobile

水果（FL Studio）作为本分类下支持系统类型最多的 DAW 之一，它同时支持在 Windows、macOS、iOS、Android 四大平台下运行，甚至还有专门的 Windows 平板版本。
FL Studio Mobile 在桌面端有着一个免费的插件版，即 FL Studio Mobile Plugin。如果想要进行电脑端的操作，可以把工程导入到电脑中的插件版进行后续编辑。
相关的信息可以参考 FL Studio Mobile 中关于 Plugin 的章节：https://www.image-line.com/fl-studio-learning/fl-studio-mobile-online-manual/html/plugins/FL%20Studio%20Mobile_FLStudioPlugin.htm

6.1.5. Auria Pro

Auria Pro 虽然在桌面端没有对应的的程序，但因为它良好的录音功能，还是存在不少用户群体。
虽然无法导出为桌面级 DAW 的工程文件格式，但 Auria Pro 提供了导出 AAF 的支持，因此把录音工程导出为通用的 AAF 格式再导入桌面级 DAW 进行编辑也算是一种变通的项目文件沟通。
如果想要以 AAF 格式导出项目，可点击 Menu -> AAF File -> Export AAF file，同时也可点击 Export AAF to DropBox 把导出的 AAF 直接存到 Dropbox 中。

信息查询网站与论坛

打破信息不对等是我的奋斗方向之一。
与其等待别人推荐，请教各种“大佬”问题，不如知道别人都是从哪里查阅的这些信息，以便自己动手丰衣足食。
下面我提供一些玩 iOS App 时必备的网站，大家可根据我写的介绍自行访问。
此外，关于 iOS 音乐类 App 的 YouTube 频道很多，在这里就不单独列出来推荐了。如果大家对相关的视频讲解或评测等有兴趣，可自行以关键词检索之。

7.1. 信息查询类网站

  • SynthyFrog，汇总了几乎所有 iOS 音乐制作类 App 的数据库，含各种 App 所支持的平台与当前价格等：https://synthyfrog.com/
  • Sweetwater iOS / iPad 板块，可查阅当前正在销售的 iOS 平台常用硬件：https://www.sweetwater.com/shop/studio-recording/ios/
  • App Sliced Music App 分类，含最新的音乐类 App 价格变化，可作为购买时的价格参考：https://appsliced.co/apps?sort=latest&l=nav&cat%5B%5D=6011

7.2. 论坛

  • Audiobus Forum，几大 Host App 中最活跃的论坛之一：https://forum.audiob.us/
  • iPad Musician Facebook Group：https://www.facebook.com/groups/Ipadmusician/
  • Reddit - iOS Music Production：https://www.reddit.com/r/iosmusicproduction/
  • Reddit - iPad Music：https://www.reddit.com/r/ipadmusic/

7.3. 微信讨论群

目前我们有一个百余人的微信群“移动平台音乐制作”，集结了国内绝大多数 iOS 音乐类 App 用户，经常会讨论软件使用、促销信息、演奏分享等内容。 DK 老师在之后也有计划组织线上/线下活动，分享更多有趣的移动平台音乐制作知识。
如果你对 iOS 音乐类 App 有兴趣，欢迎扫码加入。因为群聊二维码存在有效期限制，请在后台回复“移动音乐制作”以获得最新的二维码（如失效可在后台留言通知我）。

接下来讲什么？

亥姆霍兹实验室作为一个研究输出类公众号，从未严格的规定分享知识的类目，但关于游戏音频的分享一定占绝大多数。
对于 iOS 而言，我之后会更多分享贴近声音/音频设计师相关的内容：

  • 使用控制器 App 提升音频工作效率
  • iOS 合成器等在声音设计中的用途
  • 使用效率类 App 进行工作流改良

对于 iOS 音乐制作有兴趣的同学欢迎进入群组，关于音乐制作的其他相关内容会在群内进行单独分享。
