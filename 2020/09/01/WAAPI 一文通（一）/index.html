人人都能用 WAAPI（一）概述
2020-09-01
字数总计: 988|阅读时长: 3 分钟

大家好，我是溪夜。
去年下半年经杨磊老师推荐，我接触到了 WAAPI（Wwise Authoring API），作为头脑不怎么灵光的非专业程序员，看到 WAMP、JSON 等陌生概念直犯晕。除了官方文档，只有零星几个视频、文章和项目可参考。我不太好意思麻烦别人，只好自己在这一年中断断续续的闷头尝试，踩了不少坑。
引用 Linus 的一句话：

In open source, we feel strongly that to really do something well, you have to get a lot of people involved. — Linus Torvalds

我认为思想只有被传播才能体现价值，为了体现开源精神，决定分享一系列 WAAPI 指南给大家。《写给声音设计师的敏捷 GUI 开发大法》系列文比较偏向对新框架的介绍，故相对简单一些。而《人人都能用 WAAPI》系列文更倾向于对官方文档的补充，会更加友好且详细。
当然，对初学者友好不代表从零讲起，人人都能用 WAAPI 只代表大家都有潜力去掌握这门优秀的技能。老规矩，前置知识环节。因为个人习惯使用 Python，所以前置知识只讨论 Python。你也可以使用 WAAPI 支持的其他语言来进行 API 调用。

本系列文章阅读需要的前置知识：

  • 会配置开发环境，掌握 Python、Anaconda、VS Code、PyCharm 等的安装使用。
  • 掌握简单的 Python 语法。
  • 对 Wwise 功能有较全面的认知，因为 WAAPI 涉及的功能比较广。
  • 对现有 Wwise 工作流有优化的设想。

为什么要用 WAAPI？

如果没接触过 WAAPI，请问你是否在 Wwise 使用过程中怀疑过以下人生？

  • 能不能少一些点击和拖拽操作？
  • 成百上千（3A 项目上万也正常）的资源导入后想死怎么办？
  • 使用 Game Sync、Event、SoundBank 时能不能智能联结逻辑？
  • 不能把 DAW 和 Wwise 联动吗？
  • 如何联动 CI 系统优化持续集成管线？
  • etc.

WAAPI 的优势

作为 Wwise 2017.1 中的重要的功能补充。WAAPI 所能做的极其丰富，从简单操作中的获取工程信息、导入音频文件、设定容器层级，到声音引擎级的发送事件、设定 Game Sync 等操作均可完成，是 Wwise 音频设计师工作流的必要一环。很明显，利用好 WAAPI，能够节省大量时间，更好的把精力放在艺术创作工作上。
在本文发布的当下，可能仍有公司未对技术音频（Game Audio Programmer / Technical Sound Designer）职位进行配备。如果碰巧你是这些公司的雇员，本文会对你很有帮助。我认为对工作中重复和可优化的步骤进行不断优化，这才是智慧的工作模式，与其等待戈多，不如自己上手。

官方的资料不够全吗？

Wwise 的官方文档绝对是目前所有音频中间件中，内容最完善且本地化最好的一个。只不过文档中 WAAPI 的部分更倾向于展现给一个有一定技术背景的学习者来阅读，所以有些许门槛。这系列文章的目的如开头所述，更多是对官方文档的补充。
官方博客中也有关于 WAAPI 几篇好文，学有余力的朋友可以抽空学习。
