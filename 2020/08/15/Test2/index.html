1. 学习与开发环境

学习路线

Searching the GitHub Issues as a last resort (search both open and closed issues)
大量 YouTube 视频**

版本支持

建议 Python 3.7.2，更高的版本有或多或少的问题。Python 2.7 代会在年底删除

IDE 建议**

有些程序元素参数过多，建议使用 PyCharm，因为可在输入代码时显示文档，参照 Using The Docstrings 章节
对于初学者，可以用 PySimpleGUIWeb 在 repl.it 运行代码，也方便教学之用

代码兼容性

PySimpleGUI 具有 Windows、Mac、Linux 全平台跨平台，无需跨平台，对于 PySimpleGUIQt 也是如此
因为 PySimpleGUI 有几个分支版本，例如 PySimpleGUIWeb 可支持 Web 页面的程序。一般来说直接改掉 import 即可更换平台，但也要注意是否会出错

3. 快速入门

两种级别的窗口支持

High Level：SG 里帮忙做了很多工作的那部分，以 popup 举例一个基本的弹出窗口，可以用来显示状态


Customized：同样一行代码也可以写出自定义的 GUI，而且这是一个控件功能有效的程序

朴素的美丽

上方的代码可读性十分糟糕，把参数都拆开写，结构立马明朗。布局三行，窗口信息一行，再加上读取窗口带来的返回值

3. 切入功能代码

High Level API Calls - Popup’s 可显示一系列的弹出式窗口

不同的 popup 方法可以显示出不同的 popup 窗口，可用作完成操作后提示结果用，某种意义上可描述为是双窗口程序的雏形（主窗口与 popup 窗口）

以 Custom window API Calls 开始创建自定义窗口

三步走创建 GUI：

  1. 在纸上先画出需要的 GUI 原型
  2. 用 SG 的行逻辑把窗口分割成行
  3. 把各个元素（控件）的名字标记在窗体上
  4. 按需书写代码，代码就是窗体本身的样子
